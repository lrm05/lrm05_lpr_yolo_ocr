# OCR 与 LPRNet 对比分析

## 1. 文档目的

本文件用于支撑项目验收项中“OCR 车牌识别算法分析”和“OCR 与之前 LPRNet 车牌识别的训练、测试指标对比分析”。

需要说明的是：

当前仓库中没有历史 `LPRNet` 模型的完整训练日志、权重文件和测试记录，因此本文件分为两部分：

1. 架构与原理对比
2. 当前已完成的 OCR 实测结果

如果后续需要形成严格的“同数据、同划分、同评测口径”的数值对比表，需要补跑一版 `LPRNet` 基线实验。

## 2. 两种方案的定位

### 2.1 LPRNet

`LPRNet` 是经典的车牌识别专用网络，通常直接针对车牌图像输出整牌字符序列。

特点：
- 轻量
- 推理速度快
- 适合标准车牌场景
- 在单一数据分布下往往能取得较好效果

### 2.2 OCR 方案

本项目当前采用的 OCR 方案为 `CRNN + CTC`。

特点：
- 面向序列文本识别
- 不依赖逐字符切分
- 更容易适配不同长度和不同字符组合
- 便于后续扩展到更多车牌种类和更复杂数据分布

## 3. 模型架构对比

### 3.1 LPRNet 架构特点

典型 LPRNet 方案通常包含：

1. 卷积特征提取
2. 时序特征聚合
3. 字符分类输出

优点：
- 结构紧凑
- 推理延迟低
- 对固定场景车牌识别较高效

不足：
- 对复杂场景、复杂字符分布的适应性有限
- 可扩展性相对弱
- 当车牌样式变化增多时，需要重新做更强的适配

### 3.2 OCR(CRNN + CTC) 架构特点

本项目 OCR 模型由两部分构成：

1. `CNN` 提取图像特征
2. `RNN/时序层 + CTC` 完成字符序列解码

优点：
- 对字符序列建模更自然
- 不要求事先切分单字符
- 适合汉字、字母、数字混合序列
- 更适合后续做多省份、多类型扩展

不足：
- 相比极轻量 LPRNet，训练与推理开销可能略高
- 如果数据不足或分布偏斜，也会出现泛化不足的问题

## 4. 本项目为什么选择 OCR 路线

本次升级任务要求：

1. 使用 `YOLOv11 + OCR`
2. 支持汉字、字母、数字混合识别
3. 支持更多车牌类型扩展
4. 提升智能交通路口场景下的识别精度

因此采用 OCR 路线有以下合理性：

1. 更符合本次任务的明确技术要求
2. 更适合后续扩展到多省份和多牌种
3. 与检测模型分离，模块化更清晰
4. 后续新增数据时，更方便持续迭代

## 5. 训练方式对比

### 5.1 LPRNet 常见训练方式

LPRNet 通常直接以裁剪后的车牌图像为输入，以车牌字符序列为监督信号进行训练。

训练特点：
- 数据准备相对直接
- 对标准车牌场景训练效率较高
- 对标签规范性要求较高

### 5.2 当前 OCR 方案训练方式

当前 OCR 训练流程为：

1. 从 CCPD 文件名解析真实车牌字符
2. 根据车牌框裁剪出车牌图像
3. 生成 `charset.txt` 与 `train/val/test_label.txt`
4. 使用 `CRNN + CTC` 进行训练

训练特点：
- 与检测模型解耦
- 容易做数据增强和数据扩容
- 更适合后续加入多省份、多场景新数据

从项目升级方向看，OCR 方案相对 LPRNet 的主要优势在于：

1. 更符合本次任务的技术路线要求
