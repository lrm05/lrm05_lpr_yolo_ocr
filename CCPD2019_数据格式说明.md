## 中科数据集CCPD2019的数据格式
---
CCPD2019（Chinese City Parking Dataset）最大特点是**所有标注信息都编码在文件名中，无独立标注文件**，文件名按固定规则用 `-` 分隔成7个字段，同时数据集按场景分目录组织。

### 一、文件名格式（核心）
#### 1. 通用结构
```
[area]-[angles]-[bbox]-[four_points]-[label]-[brightness]-[blur].jpg
```
示例：`025-95_113-154&383_386&473-386&473_177&454_154&383_363&402-0_0_22_27_27_33_16-37-15.jpg`

#### 2. 逐字段解析
| 字段 | 示例值 | 含义 | 解析规则 |
|---|---|---|---|
| 1. area | 025 | 车牌区域占比 | 百分比（000–100），值越大车牌在图中占比越高 |
| 2. angles | 95_113 | 水平/垂直倾斜角 | 水平角_垂直角（单位：°），反映车牌旋转/倾斜程度 |
| 3. bbox | 154&383_386&473 | 矩形框坐标 | 左上(x1&y1)_右下(x2&y2)，`&`分隔坐标，`_`分隔两点 |
| 4. four_points | 386&473_177&454_154&383_363&402 | 四边形4个角点 | 顺序：右下→左下→左上→右上，`_`分隔4点，每点x&y |
| 5. label | 0_0_22_27_27_33_16 | 车牌字符编码 | 7位（蓝牌）/8位（绿牌），用`_`分隔，对应字符映射表 |
| 6. brightness | 37 | 亮度等级 | 0–100，值越大越亮，用于区分白天/夜间/逆光 |
| 7. blur | 15 | 模糊等级 | 0–100，值越大越模糊，用于区分清晰/模糊/运动模糊 |

#### 3. 字符编码映射（label字段）
label字段是数字序列，需对照映射表转成真实车牌：
- **省份映射（第1位）**：0=皖、1=京、2=津、3=沪、4=渝、5=冀、6=晋、7=蒙、8=辽、9=吉、10=黑、11=苏、12=浙、13=皖、14=闽、15=赣、16=鲁、17=豫、18=鄂、19=湘、20=粤、21=桂、22=琼、23=川、24=贵、25=云、26=藏、27=陕、28=甘、29=青、30=宁、31=新
- **字母/数字映射（第2–7/8位）**：0=A、1=B、2=C、3=D、4=E、5=F、6=G、7=H、8=J、9=K、10=L、11=M、12=N、13=P、14=Q、15=R、16=S、17=T、18=U、19=V、20=W、21=X、22=Y、23=Z、24=0、25=1、26=2、27=3、28=4、29=5、30=6、31=7、32=8、33=9

示例：`0_0_22_27_27_33_16` → 0（皖）、0（A）、22（Y）、27（3）、27（3）、33（9）、16（S） → **皖AY339S**

### 二、数据集目录结构
CCPD2019按场景分多个子目录，便于针对性训练/测试：
```
CCPD2019/
├── ccpd_base/       # 基础数据集（20万+，正常场景）
├── ccpd_blur/       # 模糊车牌（2万+）
├── ccpd_db/         # 亮度不均/逆光（2万+）
├── ccpd_fn/         # 远近/遮挡（2万+）
├── ccpd_rotate/     # 水平旋转（20–50°，1万+）
├── ccpd_tilt/       # 垂直倾斜（15–45°，1万+）
├── ccpd_weather/    # 雨雪雾天气（1万+）
└── splits/          # 划分文件（train.txt/val.txt/test.txt）
```

### 三、图像规格
- 尺寸：固定 **720（宽）× 1160（高）** 像素
- 通道：3通道（RGB）
- 数量：总计约 **25万+ 张**，覆盖蓝牌，无绿牌（绿牌在CCPD2020）

### 四、与PaddleOCR适配要点
1. **检测任务**：从文件名提取bbox/four_points，转成PaddleOCR检测标注格式（`图像路径\t[{"transcription":"车牌","points":[[x1,y1],...]}]`）
2. **识别任务**：裁剪bbox区域，提取label字段转成真实车牌，生成识别标注（`裁剪图路径\t车牌文本`）
3. **字符集**：仅保留车牌相关字符（31省+26字母+10数字），减少干扰

ccpd_paddleocr/

关键适配点

    字符映射：严格按照 CCPD2019 的编码规则，将数字编码转为真实车牌字符（如 0→皖、22→Y、27→3）；
    标注格式：完全符合 PaddleOCR 训练要求，可直接用于检测 / 识别模型微调；
    数据增强友好：保留了不同场景子目录（base/blur/weather/fn），便于后续按场景验证模型效果；
    有效性过滤：自动过滤解析失败、字符数不足 7 位的无效样本，保证标注质量。

总结

    该脚本一键完成 CCPD2019 数据集的解析、格式转换、裁剪和划分，直接输出 PaddleOCR 训练所需的标注文件和车牌图像；
    输出的 plate_charset.txt 是车牌专属字符集，可直接用于 PaddleOCR 识别模型的配置；
    生成的标注文件可无缝接入你之前的 PaddleOCR 微调流程，无需额外格式转换。
